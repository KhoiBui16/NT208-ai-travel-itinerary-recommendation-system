Tổng Quan Bài Toán
Mục tiêu: Triển khai agent trên nền tảng quản lý bài báo khoa học giúp user truy vấn dữliệu
bằng ngôn ngữtựnhiên.
Tài liệu tham khảo:
LangChain SQL Agent:
https://docs.langchain.com/oss/python/langchain/sql-agent
Quy trình hoạt động của agent:
1 Fetch available tables và schemas
2 Quyết định tables liên quan đến câu hỏi
3 Generate SQL query dựa trên schemas
4 Double-check query với LLM
5 Execute query và trảkết quả
6 Correct mistakes nếu có lỗi
7 Formulate response từresults

Design System
Agent layer (LangChain):
Reasoning + policy (prompt), orchestration, retry/timeout, tracing.
Không chứa credentials DB, không chạy SQL trực tiếp.
Tool layer (MCP/FastMCP):
Enforce read-only, allowlist tables/columns, limit rows, redact fields.
Trảvềoutput structured (JSON): success / error, row_count, rows_preview,
sql_executed.
Why: policy + validation nằm ởtool layer thì agent “có bịa” cũng không phá DB.

Schema-Aware Prompting (Semantic Mapping)
Problem: User hỏi “có bao nhiêu bài viết thuộc xửlý ngôn ngữtựnhiên”
Nhưng database chỉlưu “NLP@UIT” (code không giống text tựnhiên)
LLM có thểkhông tựđộng map “NLP” ←→“xửlý ngôn ngữtựnhiên”
Solution: Detailed field comments trong schema definition
Thêm comment vào mỗi field giải thích:
Mục đích (semantic meaning)
Cách dữliệu lưu trữ(format, examples)
Mapping nếu có (vd: “NLP@UIT = xửlý ngôn ngữtựnhiên”)
How it works (LLM perspective):
1 Agent gọi sql_db_schema →nhận schema + comments
2 LLM đọc comment, tựngầm map “xửlý ngôn ngữtựnhiên” ←→NLP@UIT
3 Generate WHERE clause đúng: WHERE research_group_code = ’NLP@UIT’

Testing (1): SQL Correctness
Mục tiêu: Kiểm chứng agent sinh SQL đúng cú pháp + cú ý (correct schema links, joins).
Test workflow:
1 Input: Test set câu hỏi (50) theo tier (A/B/C)
Tier A: 2-table join (simple)
Tier B: 3–4 joins + GROUP BY (intermediate)
Tier C: subquery / complex (advanced)
2 Validation:
Execution success rate: % câu query chạy không lỗi DB
Schema compliance: SQL dùng đúng table/column (allowlist check)
Join correctness: khi 2+ tables, kiểm chứng join path qua FK (pattern match)
